# Bookspire — 서비스 개요

> 완독을 해야만 영감을 얻는 건 아니잖아요?
> — 한줄에서 오는 영감, 비로소 시작되는 정독

## 어떤 서비스인가

책 한 권을 다 읽지 않아도, 하루 한 페이지에서 만난 인상 깊은 구절을 **한 줄로 남기고 공유**하는 서비스.

우리는 책을 완독하지는 않아도, 책마다 유명한 구절 하나쯤은 기억한다. Bookspire는 그 지점에서 출발한다 — 독서의 단위를 "완독"이 아니라 "한 구절"로 낮춰서, 기록과 공유의 부담을 없앤다.

## 핵심 원칙

- **완독 부담 제거** — 한 페이지만 읽어도 기록할 수 있다.
- **쓰기 부담 최소화** — 입력은 딱 두 개: 책 구절 + 책 제목. 닉네임과 작성 날짜는 자동 처리.
- **단일 화면** — 읽기(피드)와 쓰기(입력)가 한 화면에 공존하는 미니멀 구조.

## 메인 화면 구조 (위 → 아래)

| 영역 | 내용 |
|------|------|
| 상단 | 서비스 이름 "Bookspire" + 카테고리 탭(책/영화) + 헤드라인 카피. 탭에 따라 문구와 무드가 바뀜 (책: 크림/그린, 영화: 하늘색~파란색 그라데이션). |
| 중앙 | 사용자들이 남긴 메모 카드 세로 나열. 인기순/최신순 토글. |
| 하단 | 항상 보이는 입력 영역: 책 구절 + 책 제목 + 저장 버튼. |

### 메모 카드 구성

- **책 구절** — 가장 크고 눈에 띄게, 세리프 폰트
- 책 제목
- 작성자 닉네임 (자동)
- 작성 날짜 (자동)
- 추천 버튼과 추천 수

### 정렬

- **인기순** — 추천 수 기반, 좋은 구절이 상단에 노출
- **최신순** — 방금 남겨진 기록 순

## 데이터 모델 (최소)

| 필드 | 입력 방식 |
|------|-----------|
| 구절 (quote) | 사용자 입력 |
| 책 제목 (bookTitle) | 사용자 입력 |
| 닉네임 (nickname) | 자동 |
| 작성 날짜 (createdAt) | 자동 |
| 추천 수 (likes) | 추천 버튼으로 증가 |

## 구현된 기능 (2026-07-06 기준)

**핵심**
- 피드(인기순/최신순 정렬 — 클라이언트 즉시 전환), 글 작성, 추천(하트) 토글
- 제목 검색: 정렬 탭 옆 돋보기 아이콘 → 입력창 펼침, 한 글자마다 실시간 필터 (띄어쓰기 무시 부분 일치 — "일의격" ↔ "일의 격")
- 책/영화 카테고리 탭 — 탭에 따라 무드(크림/그린 ↔ 블루 그라데이션), 카피, 로고(Bookspire ↔ Cinespire) 전환
- 구절 입력: 여러 줄 지원(엔터 = 줄바꿈), 200자 제한 + 카운터, 자동 높이
- 닉네임 자동 생성: 수식어 16 × 명사 18 조합 + 숫자 (localStorage에 보관, 브라우저당 고정)
- 추천 여부도 localStorage 기억 (로그인 없음 — 1인 1추천은 서버 강제 아님)

**관리자**
- `?admin=<ADMIN_KEY>`로 접속하면 해당 브라우저에 삭제 버튼 표시 (2단계 확인)
- 삭제 API는 `x-admin-key` 헤더 검증, 키는 Vercel 환경변수 `ADMIN_KEY`

**UX 폴리시**
- 탭 전환 배경 크로스페이드, 로고·카피 페이드인, 스켈레톤 로딩
- 저장 시 "기록했어요" 토스트 + 새 카드 하이라이트, 하트 팝 애니메이션
- 입력 상자 포커스 시 테마색 테두리 강조
- 폰트: 한글 세리프 고운바탕 + 라틴 Newsreader, 워드마크 Cormorant Garamond

**운영**
- 방문자 카운터: 접속마다 +1 (중복 포함, KST 일별), 우상단에 "오늘 N번의 발걸음" 표시
- Vercel Analytics (대시보드 분석용)
- OG 메타태그 + 공유 미리보기 이미지 (카톡 등 링크 공유 시 카드 표시)
- 모바일: safe-area, 키보드 대응, dvh 뷰포트
